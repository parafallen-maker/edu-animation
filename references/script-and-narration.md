# 脚本与旁白指南

## 视频结构模板

```
┌─────────────────────────────────────────┐
│  1. Hook（开场钩子）        10-15%      │
│     提出一个问题或展示一个矛盾           │
├─────────────────────────────────────────┤
│  2. Problem（问题展开）      15-20%     │
│     加深理解，建立知识基础               │
├─────────────────────────────────────────┤
│  3. Core Content（核心内容）  40-50%    │
│     主要知识点，分 3-5 个场景            │
├─────────────────────────────────────────┤
│  4. Summary（总结回顾）      10-15%     │
│     回到开场问题，给出解答               │
├─────────────────────────────────────────┤
│  5. CTA（结尾引导）         5%          │
│     点赞/关注/下一步                      │
└─────────────────────────────────────────┘
```

## 叙事技巧

### 入场角度（Entry Angle）
- **提问式**："你有没有想过，为什么……？"
- **场景式**："想象一下，如果你家的灯泡能上网……"
- **挑战式**："大多数人认为 IoT 就是智能家居，但远远不止。"
- **故事式**："2018 年，一个冰箱参与了全球最大规模的 DDoS 攻击。"

### 核心隐喻
选一个贯穿全片的隐喻：
- IoT = "万物互联的网络" → 用神经网络/蜘蛛网类比
- 数据流 = "河流" → 用水流可视化

### 知识脚手架（Knowledge Scaffolding）
先建基础，再搭上层：
1. 观众已知的 → 互联网
2. 延伸一步 → 把物品连上网
3. 核心概念 → IoT 架构（感知→网络→平台→应用）
4. 实际影响 → 应用场景

### 情感曲线
```
兴趣 → 好奇 → 理解 → 惊叹 → 满足
 ↑      ↑      ↑      ↑      ↑
Hook  Problem Core  "原来如此" Summary
```

## 写作技巧

### 短句优先
```
❌ 物联网技术是一种将各种物理设备通过互联网连接起来实现智能化管理和控制的先进技术体系。
✓ 物联网，就是让万物上网。把灯泡、冰箱、手表连到互联网上，让它们能互相通信。
```

### 强调标记
```
**关键词** — 重读（TTS 会自然加强语气）
*轻声* — 柔和表达
[.]   — 短暂停顿（~0.3s）
[..]  — 中等停顿（~0.5s）
[...] — 长停顿（~0.8s）
[PAUSE] — 戏剧性停顿（~1.2s）
```

### 节奏控制
- Hook：快节奏，短句密集
- Core Content：中速，概念间有停顿
- Summary：放慢，回顾时留白
- CTA：简洁有力

## TTS 注意事项

### edge-tts Voice 选择
```bash
zh-CN-YunxiNeural    # 男声，年轻，适合科技/教育
zh-CN-XiaoxiaoNeural # 女声，标准，适合通用
zh-CN-YunjianNeural  # 男声，成熟，适合纪录片风格
```

### 标点影响停顿
- `，`（逗号）：~0.3s
- `。`（句号）：~0.5s
- `！`（叹号）：~0.3s + 语调上扬
- `？`（问号）：~0.5s + 语调上扬
- `、`（顿号）：~0.1s
- 省略号 `……`：~0.8s

### 语速估算
- 中文：~4 字符/秒（标准语速）
- 60 字旁白 ≈ 15 秒场景
- 120 字 ≈ 30 秒

## Phase 1.5 Self-Review Checklist

- [ ] 核心信息一行能说清
- [ ] 学习目标明确（观众看完能记住什么？）
- [ ] 每个场景的旁白完整，无占位符
- [ ] 入场角度有趣，能抓住注意力
- [ ] 有贯穿全片的核心隐喻
- [ ] 知识按依赖顺序排列（不跳跃）
- [ ] 情感曲线有起伏（不是一直平铺直叙）
- [ ] 总字数与目标时长匹配
- [ ] 无专业术语未解释的情况
- [ ] 结尾回答了开场提出的问题
